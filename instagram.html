Instagram Direct Messaging

Project Goal

To implement Instagram's voice and animated media messaging features into Facebook's Hack framework.

Project Overview

I implemented inbox loading for voice and animated media messages in the Facebook framework. I added components to the existing system and worked on merging the Instagram and Facebook platforms to assist in Meta's current infra unification movement.

Product Overview (Instagram Direct Messaging)

Instagram currently hosts a variety of message options on their Django framework. However, it less efficient than Facebook's Hack framework, and it also falls behind on keeping up to date with current user privacy rules. Therefore, the current loading feature is outdated.

Tools Used

- Hack/HHVM
- Django

The Flow

Before:

The user refreshes their inbox, sending a request to my team.

The request is sent to the Facebook framework.

The Facebook framework fetches the message data from the database.

The Facebook framework returns this data to the Instagram framework.

The Instagram framework renders the message into a format usable by the interface.

The message is displayed to the user.

After:

The user refreshes their inbox, sending a request to my team.

The request is sent to the Facebook framework.

The Facebook framework fetches the message data from the database.

The Facebook framework renders the message into a format usable by the interface.

The Facebook framework returns this data to the Instagram framework.

The message is displayed to the user.

The Result

While this seems like a small change, this makes a big impact on further merging Meta's many messaging platforms and is much more efficient than the previous model. Because Facebook is built to run more efficiently, keeping all processing on the Facebook framework increases efficieny across the board.
